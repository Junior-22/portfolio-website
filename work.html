Angular: myFlix-angular

This single-page application is built using Angular and utilizes the same serve-side code (RESTful API and database) as the React movie app. It was built to explore the differences between building the client-side with the MEAN compared to the MERN stack.

Challenges: Learning TypeScript & Angular, understanding the structure.

Implementation: Built using the MEAN stack. Applied project management techniques like Kanban Board, time estimation in story points and user flows. Configuration via Angular CLI. Implementation of user validation and security measures (JWT). Applied double-binding, directives, dependency injection, interpolation, attribute and structural directives, and routing. Visualized the interfaces with Angular material. Documentation with TypeDoc.

Meet-App

A serverless, progressive web app (pwa), which gives users access to international dev gatherings worldwide and allows filtering by location and number.

Challenges: Apply TDD approach, use testing libraries and solve dependency issues with React 18. Integrate Google Calendar API, use AWS lambda, fulfill pwa requirements, implement alerts using onLine/offline state, visualize data with JS chart library.

Pokemon App

I built a small Pokedex web app with Javascript, HTML, CSS, jQuery, and the Bootstrap framework. The code has been formatted according to ESLint rules. Users can search through a collection of Pokemons that is loaded from an external API. Images and information about a specific Pokemon is displayed in a modal view by clicking on the Pokemons name.

Challenges: Fetch and use a third-party API, target the correct endpoints, use the provided HTTP protocols. Navigate within Bootstrap.

Implementation: Connected the app to an external API. Developed the app to be user-friendly, fully-responsive, as well as optimised for all screen sizes and browsers (incl. polyfills for fetch and promise). Rules of accessibility have been followed.

Server-side: movie_api

API built for a movie application which allows users to access a database to receive information on movies. Also, the user can get background information on the movies. The user can sign up, update personal information and save/delete favorite movies to/from his/her personal profile. It’s written in Node.js and it's packages are stored in a database built using MongoDB.

Challenges: Build on top of a non-relational database (MongoDB) to have experience with working with such a database.

Implementation: Designed database schemas, created database with CRUD in SQL, documented endpoints using Express, implemented CORS.


JavaScript Node MongoDB Mongoose RESTful API Express Postman
